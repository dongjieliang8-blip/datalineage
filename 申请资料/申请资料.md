## 04 字段文本

我构建了一个名为 **DataLineage** 的多 Agent 协作数据血缘追踪与影响分析流水线系统，基于 Claude Code 开发、MiMo API 驱动。该项目解决的核心痛点是：企业数据资产日益庞大，表与表、列与列之间的依赖关系错综复杂，Schema 变更影响范围难以评估，数据合规审计（GDPR/CCPA）缺乏自动化工具。DataLineage 通过 4 个角色分工明确的 AI Agent 实现血缘扫描→依赖映射→影响分析→合规检查的完整自动化闭环。

核心逻辑流采用长链推理架构：第一层 Lineage Scanner Agent 对数据源进行深度扫描，解析 SQL/ETL 配置/API 定义，提取表级和列级血缘关系，输出数据流转图和依赖边列表；第二层 Dependency Mapper Agent 构建完整的依赖关系图谱，识别直接依赖、间接依赖和循环依赖，标注数据类型转换和聚合操作，输出拓扑排序后的依赖层级；第三层 Impact Analyzer Agent 模拟 Schema 变更（新增/删除/修改列），分析下游影响范围，评估影响程度（P0-P3），输出影响报告和迁移建议；第四层 Compliance Checker Agent 检查数据治理合规性，识别 PII 字段、敏感数据暴露风险、数据留存策略违规，输出合规审计报告。四个 Agent 间通信全部采用结构化 JSON，形成可追溯、可审计的分析链路。

项目使用 Python 构建，CLI 基于 Click + Rich 实现终端可视化。单次完整流水线运行消耗约 200-400 万 Token。目前已在个人实验室环境中投入使用，Schema 变更影响评估效率提升约 80%。
